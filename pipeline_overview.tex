\noindent This project turns raw Medicare claims and enrollment files into spouse-level event-time panels around one spouse's first heart attack or stroke, then reuses those constructed files for the main event studies, heterogeneity analyses, appendix robustness checks, annual spending tables, and a separate day-21 SNF cost-sharing RD. Format below is script | purpose | key inputs | key outputs | main links; abbreviations are IE=\nolinkurl{indexevents.dta}, IB=\nolinkurl{indexevents_base.dta}, IMS=\nolinkurl{indexevents_mergedspouses_eligible.dta}, WP=\nolinkurl{weekpanel.dta}, REM=\nolinkurl{responseevents-MEDPAR.dta}, REMDS=\nolinkurl{responseevents-MDS.dta}, Mort=\nolinkurl{mortality.dta}, and RD=\nolinkurl{RDdata.dta}.

\smallskip
Scripts 1--3 create the reusable event-date, response-event, and spouse-week analysis files that everything else builds on. The main paper then branches from WP into baseline event studies, health and spending tables, and a prediction extension, while Script 11 creates the separate RD sample that feeds the cost-sharing tables and appendix splits. Appendix scripts mostly rerun the same design under alternative sample rules, horizons, or subgroup definitions rather than constructing new raw-data pipelines.
